## Use Case 002 - Shoot Cannon

- **Primary Actor:** Player
- **Description:** The player aims and fires the cannon using the mouse.
- **Preconditions:** The player has clicked start and is in the main game loop. Canvas is loaded, timer is running.
- **Main Success Scenario:**
  1. Player moves the mouse to a `location` on the `Canvas` and `click`s.
  2. The `Cannon` `rotates` to align with the `clickLocation`.
  3. A `CannonBall` `spawn`s and `travel`s in a straight line from the `baseLocation` of the `Cannon` to the `clickLocation`.
    - **3a.** The game plays a firing sound (`cannon_fire.wav`).
  4. The `shotCounter` increments: `+=1`.
  5. Extension Point: `CannonBall` makes `contact`.
- **Extensions:**
  - **5a. Hit Target:** The `CannonBall` makes `contact` with a `Target`. 
    - `target_hit.wav` plays. 
    - `3` seconds are added to the `totalTime`. 
    - `Score` increments by `+=1`.
    - `targetCounter` decriments `-=1`.
    - `CannonBall` `despawn`s
  - **5b. Hit Blocker:** The `CannonBall` makes `contact` with the `Blocker`. 
    - `blocker_hit.wav` plays. 
    - `3` seconds are removed from the `totalTime`.
    - `CannonBall` `despawn`s
  - **5c. Miss Target:** The `CannonBall` makes `contact` with an edge of the `Canvas` 
    - `CannonBall` `despawn`s
